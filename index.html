NailRepair

Brief

This is an e-commerce Single-Page-Application, where users can create accounts, look for items to add to their basket and securely complete their checkout.

Front-end (React)

Front-end is made using the React library, with redux used as the state container.

React-Router is used to handle all the routes, turning this into a Single-Page-Application.

All CSS is done from scratch without using any framework.

Back-end (Express.js)

The back-end is a RESTful API managed by the express framework.

The login authentication is handled using the passport library. All passwords managed in this application are hashed for safety.

Database (PostgreSQL)

This web application uses a PostgreSQL database.

Any calls made to the database is handled using the node-postgres module. Parameterized queries are used to reduce SQL injection vulnerabilities.

Note: Search bar and checkout are currently not functional.
Currently working on setting up a server and database to make all features of the website accessible

Restaurant

Brief

Website for a restaurant, with the ability to book tables.

Main feature: when booking a table, it displays the number of available seats to show you how busy the restaurant is.

Other features:

  • You can view the menu. Data for the menu is store on the database making it easy to update.
  • About us page containing the basic information about the restaurant.
  • Locations page containing details of the restaurants locations. Data for each restaurant is again stored in a database, making it easy to update.

Front-end (React)

Same as the e-commerce project, front-end is made using the React library, with redux used as the state container.

React-Router is used to handle all the routes, turning this into a Single-Page-Application.

All CSS is done from scratch without using any framework.

Back-end (Express.js)

The back-end is a RESTful API managed by the express framework.

Database (PostgreSQL)

This web application uses a PostgreSQL database.

Any calls made to the database is handled using the node-postgres module. Parameterized queries are used to reduce SQL injection vulnerabilities.
